# Onboarding: Your First Project

When you start using Scope, your first project lives under **Internal**.

This is intentional.

---

## Why Internal comes first

Every business has internal work.

That includes things like:
- Personal projects
- Internal tools
- Planning and setup
- Experiments and refactors
- Getting oriented in a new system

This work is real, even if it isn’t billed.

---

## Your first project

When we ask you to create your first project, it’s under **Internal** by default.

This project is:
- Yours
- Non-invoiced
- Fully tracked
- Included in reports

Think of it as:
> “Getting set up — this one’s on us.”

---

## What to do next

You can:
- Track time
- Add tasks
- Upload documents
- Explore reports

When you’re ready, you can create a client project just as easily.

Nothing about this project is temporary or fake.
It’s real work, in a real system.

---

## The idea to remember

Internal work isn’t a workaround.

It’s how work actually starts.
