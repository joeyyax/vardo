# Task Workflow vs Project Lifecycle

This document clarifies the distinction between task status and project lifecycle state.

They serve different purposes and must not be conflated.

---

## Project Lifecycle

Project lifecycle represents **where we are in the engagement**.

Examples:
- Getting Started
- Proposal
- Agreement
- Onboarding
- Active
- Ongoing
- Completed
- Archived

Lifecycle transitions are:
- Infrequent
- Intentional
- Gated
- Provider-owned

Changing a project state changes the rules of work.

---

## Task Workflow

Task workflow represents **execution inside an active project**.

Task statuses include:
- Todo
- In Progress
- Review
- Done

Task status changes are:
- Frequent
- Operational
- Local to the project
- Provider-owned

Tasks do not change the rules of work.

---

## Key Rules

- Tasks can move freely without changing project state
- Projects do not advance because tasks move
- Work does not begin until the project is Active
- Lifecycle transitions never happen automatically

---

## Anti-Patterns to Avoid

- Using tasks to simulate onboarding
- Treating “In Progress” tasks as project start
- Running multiple lifecycles inside one project
- Letting task volume dictate project state

---

## Summary

Projects define **intent and boundaries**.  
Tasks represent **execution**.

They are related, but not interchangeable.
