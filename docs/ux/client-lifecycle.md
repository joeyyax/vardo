The Client Lifecycle Phases (canonical definition)

This is both product spec and shared mental model.

You don’t have to show this verbatim to clients — but everything they see should map to it.

⸻

Phase 0 — Inquiry

Internal state: inquiry
Client-facing: none

Purpose:
	•	Initial contact
	•	Fit check
	•	No commitments

No dashboard yet.

⸻

Phase 1 — Getting Started

State: getting_started

This is the first visible phase for the client.

Client sees:
	•	Welcome message
	•	How We’ll Work Together
	•	A simple status:
We’re getting set up and aligning on next steps.

Nothing actionable yet except reading.

Goal:
	•	Orientation
	•	Trust
	•	Calm

⸻

Phase 2 — Proposal

State: proposal

Client sees:
	•	Proposal document
	•	Clear pricing and scope
	•	One primary action: Review proposal

Status copy example:

Review the proposed work and pricing.

Goal:
	•	Alignment
	•	Decision-making

No work has started yet.

⸻

Phase 3 — Agreement

State: agreement

Client sees:
	•	Contract
	•	Any relevant addenda
	•	One primary action: Accept agreement

Status copy example:

We’re locking in the details so work can begin.

This is the actual start gate.

⸻

Phase 4 — Onboarding

State: onboarding

Client sees:
	•	Onboarding checklist
	•	Access requests
	•	Setup tasks

Status copy example:

We’re gathering access and setting up the project.

This phase ends when:
	•	Required access is complete
	•	Initial setup is done

⸻

Phase 5 — Active

State: active

Client sees:
	•	Tasks
	•	Docs
	•	Discussions
	•	Progress

Status copy example:

Work is underway.

This is the longest phase and the heart of the relationship.

⸻

Phase 6 — Ongoing (optional)

State: ongoing

Used for:
	•	Retainers
	•	Maintenance
	•	Long-term support

Client sees:
	•	Ongoing activity
	•	Capacity / hours (if relevant)
	•	Recent updates

Status copy example:

Ongoing support and improvements.

⸻

Phase 7 — Offboarding (only if needed)

State: offboarding

Client sees:
	•	Request application data
	•	Migration checklist
	•	Migration assistance (optional)

Status copy example:

We’re wrapping things up and preparing your data.

This phase should feel controlled and respectful, not abrupt.

⸻

Why this will wow clients

Because:
	•	They always know where they are
	•	They’re never overwhelmed with docs
	•	They see progress even before work starts
	•	The process feels intentional, not ad hoc
	•	You feel bigger than “just a freelancer” without pretending to be a giant org

Most freelancers jump straight from “sure” to “send me access.”
You’re giving clients a shared map instead.
