# Active Project UX

This document defines the user experience once a project has completed onboarding and active work has begun.

The Active stage is the default day-to-day experience for most projects.

---

## Entry Point

### When this stage begins

- Onboarding checklist is completed
- Project state advances to **Active**

---

## Project Page — Active Stage

### Status

> **Active**  
> Work is underway.

### Visible Elements

- Lifecycle timeline
  - Onboarding (completed)
  - Active (current)
  - Ongoing (optional, locked)

- Primary workspace areas:
  - Tasks
  - Project Docs
  - Discussions / Updates

---

## Primary Actions

Primary actions depend on project type, but typically include:

- **Create task / request**
- **Comment or discuss**
- **View project status**

There is no single forced action in this stage.

---

## Tasks

### Purpose

Tasks are the primary unit of work.

Tasks may represent:
- Features
- Fixes
- Requests
- Research
- Follow-ups

### UX Characteristics

- Tasks are scoped to the project
- Tasks have clear status (e.g. open, in progress, completed)
- Clients can submit tasks directly (if allowed)
- Task history is preserved

Tasks should feel lightweight, not ticket-heavy.

---

## Project Docs

### Purpose

Project Docs act as a living record.

Docs include:
- Proposal (accepted)
- Agreement and addenda
- Onboarding checklist (completed)
- SOWs or change orders
- Notes or summaries generated during work

Docs are:
- Generated by the system
- Added as work progresses
- Read-only by default

Clients do not manage files manually.

---

## Discussions & Updates

### Purpose

Provide context without clutter.

- Questions
- Clarifications
- Status notes
- Decisions

This replaces scattered email threads.

---

## Status Visibility

At all times, the client should see:
- Current project phase
- Recent activity
- What’s currently in progress

Clients should never wonder:
> “Is anything happening?”

---

## Constraints

- Active work cannot begin without onboarding
- Agreements and proposals remain immutable
- Scope changes flow through change orders, not ad-hoc edits

---

## Summary

The Active stage:
- Is calm and transparent
- Centers around tasks and progress
- Makes work visible without micromanagement

This is where trust compounds.
