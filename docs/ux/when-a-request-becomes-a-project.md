# When a Request Becomes a Project

Some requests fit neatly into ongoing work. Others deserve their own project.

---

## Tasks vs Projects

A request stays a task if it:
- Fits within the current scope
- Doesn’t change timelines
- Doesn’t affect pricing

A request becomes a project if it:
- Adds new scope
- Introduces a new timeline
- Requires a separate agreement

---

## Why This Matters

Separating projects:
- Keeps expectations clear
- Prevents confusion
- Makes progress easier to track

Even when work touches the same site, different goals deserve their own space.

---

## Summary

Projects create clarity — not complexity.
