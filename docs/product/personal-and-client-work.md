# Personal Work & Client Work

This system is designed to support intentional work — whether that work involves clients or not.

Client-facing features are additive.  
They do not define the core workflow.

---

## Core Idea

At its heart, this product is a system for:
- Defining intent
- Structuring work
- Executing deliberately
- Closing loops cleanly

Client collaboration is one mode of operation — not a prerequisite.

---

## Personal Projects

Personal projects use the same core primitives:
- Projects define goals and constraints
- Tasks execute inside those boundaries
- Time, expenses, notes, and documents stay connected
- Lifecycle still matters (start, active, pause, done)

For personal work:
- There is no client portal
- There are no approvals
- Visibility is internal only

The system still provides clarity, structure, and momentum.

---

## Client Projects

Client projects introduce:
- External visibility
- Explicit agreements
- Approvals and onboarding
- Collaboration surfaces (comments, portals, requests)

These features:
- Enhance collaboration
- Protect scope and expectations
- Do not alter the underlying workflow model

The same rules apply:
- Projects define rules
- Tasks execute within them
- Work starts intentionally

---

## Design Constraint

**The system must remain useful for solo, personal projects.**

This means:
- No feature should require a client to exist
- No workflow should depend on external approval to function
- Client-facing features must layer on top of the core, not replace it

If a feature only makes sense with a client present, it must be optional.

---

## Why This Matters

This constraint ensures the product:
- Feels coherent instead of fragmented
- Works equally well for freelancers and small teams
- Avoids becoming “client management software”
- Remains calm, flexible, and intentional

It also reflects real life:
- Not all work is client work
- Not all projects need collaboration
- The same discipline applies either way

---

## Summary

This is a system for doing work well.

Sometimes that work involves clients.  
Sometimes it doesn’t.

The workflow stays the same.
