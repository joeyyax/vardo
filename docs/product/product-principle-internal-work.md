# Product Principle: Internal Work

Internal work is not an edge case.

It is a core part of how real work happens.

---

## The Principle

> Internal work is non-invoiced, not invisible.

Any feature that hides internal work is incorrect by default.

---

## Design Implications

When designing features:
- Internal work must behave like real work
- Exceptions should be billing-related only
- Reporting should remain honest
- UX should never imply internal work is “lesser”

---

## Guardrail

If a decision:
- Makes internal work harder to track
- Removes it from totals
- Treats it as second-class

That decision should be reconsidered.

---

## Summary

Scope models reality, not convenience.

Internal work is reality.
