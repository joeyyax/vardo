# Configuration

## Environment Variables

Host uses two environment files:

- **`.env`** -- Non-secret development defaults, committed to the repository.
- **`.env.prod`** (production) or **`.env.local`** (development) -- Secret overrides, never committed.

### Core Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://host:host@localhost:5433/host` |
| `REDIS_URL` | Redis connection string | `redis://localhost:6379` |
| `BETTER_AUTH_SECRET` | Secret key for session signing and encryption | `host-dev-secret-change-in-production` |
| `NEXT_PUBLIC_BETTER_AUTH_URL` | Public URL of the Host instance | `http://localhost:3000` |
| `NEXT_PUBLIC_APP_URL` | Public URL of the Host instance (used by the frontend) | `http://localhost:3000` |

### Vardo Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `HOST_DOMAIN` | Dashboard domain (production only) | -- |
| `HOST_BASE_DOMAIN` | Base domain for auto-generated app subdomains | `localhost` |
| `HOST_EXPOSE_PORTS` | Enable direct port exposure for apps | `true` (dev) |
| `HOST_PROJECTS_DIR` | Directory for deployment data inside the container | `/var/lib/host/projects` |

### Encryption

| Variable | Description | Default |
|----------|-------------|---------|
| `ENCRYPTION_MASTER_KEY` | 256-bit hex key for encrypting secrets (env vars, deploy keys). Generate with `openssl rand -hex 32` | -- |

### Email (Mailpace)

| Variable | Description | Default |
|----------|-------------|---------|
| `MAILPACE_API_TOKEN` | Mailpace API token for sending emails (magic links, notifications) | -- |
| `EMAIL_FROM` | Sender address for outgoing emails | `Host <deploy@example.com>` |
| `EMAIL_REPLY_TO` | Reply-to address | `you@example.com` |

### GitHub App

| Variable | Description | Default |
|----------|-------------|---------|
| `GITHUB_APP_ID` | GitHub App ID | -- |
| `GITHUB_APP_SLUG` | GitHub App slug (URL name) | -- |
| `GITHUB_CLIENT_ID` | OAuth client ID | -- |
| `GITHUB_CLIENT_SECRET` | OAuth client secret | -- |
| `GITHUB_WEBHOOK_SECRET` | Secret for verifying GitHub webhook signatures. **Required** — the webhook endpoint returns 500 if unset. Must be unique; never share with `BETTER_AUTH_SECRET`. | -- |
| `GITHUB_PRIVATE_KEY` | Base64-encoded PEM private key for the GitHub App | -- |

### Monitoring (Production)

| Variable | Description | Default |
|----------|-------------|---------|
| `CADVISOR_URL` | URL for the cAdvisor container metrics endpoint | `http://cadvisor:8080` |
| `LOKI_URL` | URL for the Loki log aggregation endpoint | `http://loki:3100` |

### Traefik (Production)

| Variable | Description | Default |
|----------|-------------|---------|
| `ACME_EMAIL` | Email for Let's Encrypt certificate registration | -- |
| `TRAEFIK_DASHBOARD_AUTH` | BasicAuth credentials for the Traefik dashboard | Generated by installer |

### Analytics (Optional)

| Variable | Description | Default |
|----------|-------------|---------|
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | Domain for Plausible analytics tracking | -- |
| `NEXT_PUBLIC_PLAUSIBLE_SRC` | Plausible analytics script URL | -- |

## Feature Flags

Feature flags control which capabilities are available system-wide. All flags default to **enabled**. Set the environment variable to `"false"` to disable a feature.

Core features (projects, apps, deployments) cannot be disabled.

| Flag | Env Var | Description |
|------|---------|-------------|
| Web UI | `FEATURE_UI` | Web dashboard for managing projects, apps, and deployments |
| Metrics | `FEATURE_METRICS` | Container CPU, memory, and network monitoring with historical charts |
| Logs | `FEATURE_LOGS` | Persistent log collection and streaming via Loki |
| Terminal | `FEATURE_TERMINAL` | Web-based terminal access to running containers |
| Environments | `FEATURE_ENVIRONMENTS` | Multiple deployment environments per app (staging, preview) |
| Backups | `FEATURE_BACKUPS` | Scheduled volume backups with S3-compatible storage |
| Cron Jobs | `FEATURE_CRON` | Scheduled command execution inside containers |

Example -- disable metrics and logs on a resource-constrained server:

```bash
FEATURE_METRICS=false
FEATURE_LOGS=false
```

Feature flags gate both UI tabs and their corresponding API endpoints.

## host.toml -- Config as Code

Place a `host.toml` file in your repository root to configure deploy-time settings. Host reads this file during deployment and applies the settings automatically.

Supported filenames (checked in order): `host.toml`, `.host.toml`, `host.config.toml`.

### Structure

```toml
[project]
name = "my-app"
displayName = "My App"
description = "A web application"
icon = "https://example.com/icon.png"
deployType = "dockerfile"
port = 3000
rootDirectory = "."

[deploy]
autoDeploy = true
branch = "main"
restartPolicy = "unless-stopped"

[[envVars]]
key = "NODE_ENV"
value = "production"

[[envVars]]
key = "LOG_LEVEL"
value = "info"

[[volumes]]
name = "data"
mountPath = "/app/data"

[[domains]]
domain = "myapp.example.com"
ssl = true

[[cron]]
name = "cleanup"
schedule = "0 2 * * *"
command = "node cleanup.js"
```

### Fields

**`[project]`** -- App metadata and build configuration.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | URL-safe app name |
| `displayName` | string | Human-readable name |
| `description` | string | App description |
| `icon` | string | Icon URL |
| `deployType` | string | `compose`, `dockerfile`, `image`, `nixpacks`, `static` |
| `port` | integer | Container port for HTTP traffic |
| `rootDirectory` | string | Subdirectory containing the app |

**`[deploy]`** -- Deployment behavior.

| Field | Type | Description |
|-------|------|-------------|
| `autoDeploy` | boolean | Auto-deploy on git push |
| `branch` | string | Git branch to track |
| `restartPolicy` | string | Docker restart policy (`unless-stopped`, `always`, `no`, `on-failure`) |

**`[[envVars]]`** -- Environment variables (array of tables).

**`[[volumes]]`** -- Persistent volumes (array of tables).

**`[[domains]]`** -- Custom domains (array of tables).

**`[[cron]]`** -- Cron jobs (array of tables).

## Template Format

Templates are TOML files in the `templates/` directory. Each file defines a deployable service with sensible defaults.

### Example: PostgreSQL

```toml
version = "1"
name = "postgres"
displayName = "PostgreSQL"
description = "Powerful open-source relational database"
icon = "https://cdn.simpleicons.org/postgresql/4169E1"
category = "database"
source = "direct"
deployType = "image"
imageName = "postgres:16"
defaultPort = 5432

[[envVars]]
key = "POSTGRES_PASSWORD"
description = "Superuser password"
required = true

[[envVars]]
key = "POSTGRES_USER"
description = "Superuser name"
required = false
defaultValue = "postgres"

[[envVars]]
key = "POSTGRES_DB"
description = "Default database name"
required = false
defaultValue = "postgres"

[[volumes]]
name = "data"
mountPath = "/var/lib/postgresql/data"
description = "PostgreSQL data files"

[[connectionInfo]]
label = "Vardo"
value = "${project.name}"
copyRef = "HOST"

[[connectionInfo]]
label = "Connection URL"
value = "postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@${project.name}:5432/${POSTGRES_DB}"
```

### Template Fields

| Field | Type | Description |
|-------|------|-------------|
| `version` | string | Template format version |
| `name` | string | Unique identifier (used as filename without `.toml`) |
| `displayName` | string | Human-readable name |
| `description` | string | Short description |
| `icon` | string | Icon URL |
| `category` | string | One of: `database`, `cache`, `monitoring`, `web`, `tool`, `custom` |
| `source` | string | `direct` or `git` |
| `deployType` | string | `image`, `compose`, `dockerfile`, `nixpacks`, `static` |
| `imageName` | string | Docker image (for `image` deploy type) |
| `gitUrl` | string | Git repository URL (for `git` source) |
| `gitBranch` | string | Git branch |
| `composeContent` | string | Inline compose file content |
| `rootDirectory` | string | Subdirectory in the repository |
| `defaultPort` | integer | Default container port |

**`[[envVars]]`** -- Default environment variables.

| Field | Type | Description |
|-------|------|-------------|
| `key` | string | Variable name |
| `description` | string | Human-readable description |
| `required` | boolean | Whether the user must provide a value |
| `defaultValue` | string | Default value if not required |

**`[[volumes]]`** -- Default persistent volumes.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Volume name suffix |
| `mountPath` | string | Mount path inside the container |
| `description` | string | Human-readable description |

**`[[connectionInfo]]`** -- Connection metadata shown in the UI.

| Field | Type | Description |
|-------|------|-------------|
| `label` | string | Display label (e.g. "Vardo", "Port", "Connection URL") |
| `value` | string | Value, supports `${...}` template expressions |
| `copyRef` | string | Optional env var name for the copy button |

## Docker Compose Profiles

The development environment uses `docker-compose.override.yml` to customize the stack for local development:

- The `host` service is disabled (use `pnpm dev` for hot reload instead).
- PostgreSQL is exposed on port `5433`.
- Redis is exposed on port `6379`.
- cAdvisor is exposed on port `8081`.
- Loki is exposed on port `3100`.
- Traefik runs with an insecure dashboard on port `8080` (no TLS, no BasicAuth).

In production, the base `docker-compose.yml` is used directly with `--env-file .env.prod`. All services communicate over internal Docker networks and only Traefik exposes ports 80 and 443 to the host.
