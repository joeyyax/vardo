# Migration Assistance

**Client:** {ClientName}  
**Project:** {ProjectName}  
**Related Agreement:** {ContractName}

This document explains how Provider can assist if Client wishes to move their application away from Provider-managed hosting.

---

## 1. Data Ownership

All application data belongs to Client.

Client may request an automated export of their application data at any time using the “Request Application Data” feature. This export is described separately and is always available regardless of migration assistance.

---

## 2. What Migration Assistance Is

Migration assistance is optional, paid help to support the transition of Client’s application to a new environment.

Migration assistance may include:
- Explaining the structure of the exported data
- Answering reasonable questions about deployment or configuration
- Coordinating timing of final exports
- Assisting with handoff to a new provider

Migration assistance does not guarantee that the application will run in a new environment without modification.

---

## 3. What Migration Assistance Is Not

Unless explicitly agreed in writing, migration assistance does not include:
- Rebuilding or re-architecting the application
- Debugging or fixing issues in a new environment
- Supporting third-party providers or platforms
- Ongoing support after migration is complete
- Guaranteeing compatibility with a new host or stack

---

## 4. Fees

Migration assistance is billed separately at Provider’s standard rates unless otherwise agreed.

Estimates may be provided but are not guarantees.

---

## 5. Timing and Coordination

Client is responsible for selecting a new hosting provider and preparing their new environment.

Provider will make reasonable efforts to coordinate timing, but migration speed depends on:
- Application complexity
- Data size
- Third-party systems
- Client responsiveness

---

## 6. Completion

Migration assistance concludes when:
- Requested data exports have been provided, and
- Any agreed-upon assistance tasks are completed.

Provider has no ongoing responsibility for the application after hosting services end unless a new agreement is in place.

---

## 7. Questions

If you’re considering a migration and aren’t sure what help you need, just ask. We can talk through options before anything changes.
