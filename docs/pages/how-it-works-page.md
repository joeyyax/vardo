# How It Works

This product is built around a simple idea:

**Work should be intentional from start to finish.**

---

## Projects define the rules

Every piece of work lives inside a project.

Projects define:
- Scope
- Pricing
- Timelines
- Expectations

Nothing important happens without a clear container.

---

## Tasks execute inside projects

Tasks are how work gets done.

They:
- Live inside projects
- Do not redefine scope
- Do not change billing
- Do not start work implicitly

Tasks move. Projects advance deliberately.

---

## A clear lifecycle

Work follows a natural sequence:

1. Proposal
2. Agreement
3. Onboarding
4. Active work
5. Ongoing or offboarding

The system respects this order.

Work does not start until it’s agreed.  
Transitions are explicit.  
Nothing is assumed.

---

## Visibility without control

Clients can:
- See progress
- Comment
- Submit requests
- Approve when needed

They do not manage the workflow.

This keeps collaboration clear without creating chaos.

---

## Automation with restraint

Automation exists to:
- Reduce busywork
- Improve consistency
- Save time

It never:
- Starts work
- Changes scope
- Surprises anyone

Human judgment stays in control.

---

## Works for personal projects too

Client collaboration is optional.

The same structure works for:
- Personal projects
- Internal initiatives
- Solo work

Client-facing features layer on when needed.

---

## The result

A system that:
- Feels calm
- Reduces decision fatigue
- Keeps work and agreements aligned
- Gets out of your way

That’s how it works.
