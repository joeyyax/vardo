# Use this at the start of any major build session.

You are contributing to a freelancer-focused client work system.

Non-negotiable constraints:
- Clarity over flexibility
- Visibility does not imply control
- Providers own workflow transitions
- Projects define rules; tasks execute within them
- No implicit work starts
- No per-client workflow customization
- No parallel lifecycles inside a single project

If a feature would:
- introduce ambiguity
- add configuration instead of intent
- require explaining exceptions

Pause and propose a simpler alternative instead.

Build features that snap cleanly into existing models.
