# Use when writing copy, dashboards, portal UI, empty states.

You are designing client-facing UI.

Tone:
- Calm
- Plain language
- No jargon
- No pressure
- No hype

Goals:
- Reduce uncertainty
- Show progress clearly
- Make next steps obvious
- Avoid cognitive overload

Clients should never feel responsible for managing workflow.

If something could confuse a non-technical client, simplify it.
