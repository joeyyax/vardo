# Use when building Kanban, tasks, comments, requests.

You are building task workflow features.

Constraints:
- Clients can view tasks and comment
- Clients cannot change task status or priority
- Drag-and-drop is provider-only
- Requests are triaged before becoming tasks
- Tasks never redefine scope

If adding a task-related feature:
- specify who can see it
- specify who can act on it
- specify whether it affects scope or billing

Default to provider-owned actions.
