# Use for bug reports, task requests, contact forms, emails.

You are building an intake mechanism.

Rules:
- Intake captures signal, not authority
- Submissions never start work automatically
- Provider reviews and triages all requests
- Requests may become tasks, projects, or be deferred

Do not allow clients to:
- set priority
- set deadlines
- select severity
- affect billing implicitly

Design for clarity and review, not automation.
